# IPC Extractor 技術手冊

## 專案概述
本工具旨在從 IPC 分類階層的 PDF 檔案中自動提取「次類」(Sub-class) 標籤及其對應的敘述內容。

## 專案結構與版本控制
為了保持儲存庫的整潔並確保協作時資料夾結構的一致性，本專案採用以下規則：
- **`data/`**：存放原始 PDF 數據。資料夾本身被追蹤，但內部的 PDF 檔案被忽略。
- **`output/`**：存放生成的 CSV/TSV 結果。資料夾本身被追蹤，但內容被忽略。

> [!NOTE]
> 以上資料夾內均包含 `.gitkeep` 文件，以確保空資料夾能被 Git 納入版本控制。

## 核心技術實現

### 1. 文本提取 (Text Extraction)
使用 `PyMuPDF` (pymupdf) 庫進行 PDF 內容讀取。
- **頁面篩選**：腳本僅掃描 PDF 前 20 頁中出現「目次-N」頁首標記的頁面，大幅準確化目次識別。
- **高效能解析**：採用 `get_text("text")` 模式快速掃描文本，不解析圖像以提升速度。
- **邊界控制**：進入目次區後，若次頁未偵測到標記則視同進入主文並自動終止。

### 2. 資料清洗 (Data Cleaning)
針對 PDF 轉換常遇到的問題進行雙重過濾：
- **頁碼清洗策略**：
    - **策略 A**：適用字母-數字型標記（如 `A-12`, `D_34`）。
    - **策略 B**：適用點線及純數字型標記（如 `......... 20` 或帶版本注記 `[2] 120`）。
- **CJK 斷行優化**：利用正則表達式自動消除中文字元（CJK）與其後接符號或大寫英文字元間的冗餘空格，確保文本流暢性。

### 3. 解析邏輯 (Parsing Logic)
採用嚴謹的狀態機 (State Machine) 模式：
- **嚴格標籤識別**：次類標籤後必須為行尾或接 2 個以上空白，避免誤抓描述中的交叉引用（如 `H01J，H05G`）。
- **防呆與煞車**：遇頁碼參考立即「煞車」(Flush)；遇「附註」或「次類索引」等主文特徵立即「斷電」(Break) 中斷。
- **跨行處理**：支持標籤與描述同行或跨行分佈。

## 資料格式

### CSV 輸出
- **路徑**: `output/ipc_subclasses.csv`
- **欄位**: `ipc_label`, `description`
- **特性**: 包含 BOM (UTF-8-SIG) 以利 Excel 直接開啟。

### PostgreSQL TSV 輸出
- **路徑**: `output/ipc_subclasses.tsv`
- **欄位**: `ipc_label`, `description` (以 Tab 分隔)
- **特性**: 移除 Tab 與換行符號，適合 `COPY` 指令快速匯入。

## 維護與擴充
若需支援「主類」(Main-class)，可於 `IPCDataExtractor` 初始化時將 `include_main_class` 設為 `True`。
